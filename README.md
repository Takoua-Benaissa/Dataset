# Text-to-Image Dataset Creation Pipeline

A fully automated pipeline that builds a **high-quality image dataset for training text-to-image generation models**.  
Each entry pairs a real photograph containing clearly visible text with:
- the **exact OCR-extracted text** present in the image
- a **rich visual description** generated by BLIP
- a **ready-to-use training prompt** of the form:  
  > *"A storefront at night, with the text "OPEN 24 HRS" clearly visible, sharp focus, high resolution photography"*

---

## Why this dataset?

Text-to-image models (Stable Diffusion, DALL¬∑E, etc.) notoriously struggle to render readable text inside generated images.  
Training or fine-tuning on a curated dataset where:
1. the text is **physically present and OCR-verified** (not just described in a caption),
2. the image is **sharp, well-lit and high-resolution**,
3. the prompt **explicitly names the text**,

‚Ä¶gives the model concrete examples of what "text clearly visible" actually looks like, significantly improving its text-rendering capability.

## Data source

This pipeline uses the **[TextCaps dataset](https://textvqa.org/textcaps/)** (Sidorov et al., 2020):

| Split | Images | Annotations |
|-------|--------|-------------|
| Train | 21 953 | [TextCaps_0.1_train.json](https://dl.fbaipublicfiles.com/textvqa/data/textcaps/TextCaps_0.1_train.json) |
| Test  |  3 289 | [TextCaps_0.1_test.json](https://dl.fbaipublicfiles.com/textvqa/data/textcaps/TextCaps_0.1_test.json) |

Images are downloaded directly from their original **Flickr URLs** (stored in the JSON files).  
No manual download from the TextVQA website is required ‚Äî the pipeline handles everything automatically.

## Quickstart ‚Äî run locally or on any Linux machine

### Step 1 ‚Äî Clone and set up the environment

```bash
git clone https://github.com/Takoua-Benaissa/Dataset_creation.git
cd Dataset_creation

python3 -m venv venv
source venv/bin/activate
pip install --upgrade pip
pip install -r requirements.txt
```

### Step 2 ‚Äî Download the annotations

```bash
mkdir -p ~/data
# Train annotations (172 MB, 21 953 images)
wget https://dl.fbaipublicfiles.com/textvqa/data/textcaps/TextCaps_0.1_train.json \
     -O ~/data/TextCaps_0.1_train.json
```

### Step 3 ‚Äî Download the images

```bash
python download_train_images.py \
    --json    ~/data/TextCaps_0.1_train.json \
    --output  ~/data/train_images/ \
    --workers 20
```

> This downloads ~21 000 images in parallel from Flickr (~15‚Äì30 min depending on connection).  
> All images are cached; re-running skips already-downloaded files.

### Step 4 ‚Äî Run the pipeline

```bash
python create_dataset_cluster.py \
    --images    ~/data/train_images/ \
    --output    ./dataset_output/ \
    --max-images 500
```

After completion you will find:

```
dataset_output/v1/

‚îú‚îÄ‚îÄ val/images/          # 50  images (10 %)
```

## Run on a SLURM GPU cluster

If you are on an HPC cluster with SLURM (e.g. ENSTA Paris cluster):

### One-time setup
```bash
bash setup.sh
```

### Download data (on login node, no GPU needed)
```bash
# Download train annotations
wget https://dl.fbaipublicfiles.com/textvqa/data/textcaps/TextCaps_0.1_train.json \
     -O ~/data/TextCaps_0.1_train.json

# Download images in background
source venv/bin/activate
python download_train_images.py \
    --json ~/data/TextCaps_0.1_train.json \
    --output ~/data/train_images/ \
    --workers 20 &
```

### Submit the job
```bash
sbatch job_creation_dataset.sh
```

The script automatically:
- detects which GPU partition is available
- selects the best image source (train > combined train+test > test)
- sets all Tesseract paths
- saves versioned output (`dataset_output/v1/`, `v2/`, ‚Ä¶)

### Monitor
```bash
squeue -u $USER
tail -f logs/dataset_<JOBID>.out
```

## Parameters

All thresholds are defined at the top of `create_dataset_cluster.py`:

| Parameter | Default | Description |
|-----------|---------|-------------|
| `MIN_RESOLUTION` | 300 px | Minimum image width and height |
| `MIN_SHARPNESS` | 150 | Laplacian variance ‚Äî rejects blurry images |
| `BRIGHTNESS_MIN/MAX` | 40 / 215 | Rejects too dark or washed-out images |
| `MIN_CONTRAST` | 30 | Std-dev of gray channel ‚Äî rejects flat images |
| `MIN_OCR_CONF` | 70 % | Minimum Tesseract per-word confidence |
| `MIN_TEXT_LEN` | 3 chars | Ignore single letters / OCR noise |
| `MAX_TEXT_LEN` | 40 chars | Upper bound on extracted text length |

To generate a larger dataset (e.g. 1 000 images) simply pass `--max-images 1000`.

## Expected output quality

With the default settings, from ~18 000 train images you can expect:

- ~**25‚Äì35 %** acceptance rate (the rest are rejected by quality or OCR filters)
- Average OCR confidence: **> 85 %**
- Text length distribution: mostly **4‚Äì20 characters** (single words to short phrases)
- All images: **256 √ó 256 px**, JPEG quality 95

Sample accepted entries:

| Text | BLIP caption | Scene |
|------|-------------|-------|
| `BANANAS` | A cork board with a bunch of fruit on it | Price tag on market display |
| `Games` | The amazon fire hd tablet | Tablet showing app store |
| `CALCULATOR` | A calculator sitting on top of a table | Office desk |
| `OPEN 24 HRS` | A neon sign on the side of a building | Storefront at night |
